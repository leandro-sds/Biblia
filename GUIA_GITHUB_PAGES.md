# 📱 Guia: Bíblia Sagrada como PWA no GitHub Pages

## O que é uma PWA e por que é melhor para você

Uma PWA (Progressive Web App) é um site que se comporta como um aplicativo:
- A pessoa abre o navegador, entra no seu link, e aparece um botão "Instalar"
- Depois de instalar, fica um ícone na tela inicial do celular, igual a qualquer app
- Funciona **offline** — depois da primeira visita, não precisa de internet
- **Compatível com TalkBack** e leitores de tela
- Não precisa de Android Studio, nem de conta na Play Store

---

## O que você vai fazer

1. Criar uma conta gratuita no GitHub
2. Criar um "repositório" (uma pasta online)
3. Enviar os arquivos da Bíblia
4. Ativar o GitHub Pages — seu site/app estará no ar!

Tudo isso pode ser feito pelo **navegador**, sem instalar nada.

---

## PASSO 1 — Criar conta no GitHub

1. Acesse **https://github.com**
2. Clique em **"Sign up"**
3. Escolha um nome de usuário (ex: `leandrosouza`)
4. Digite seu e-mail e uma senha
5. Confirme o e-mail que eles enviam

> Seu nome de usuário vai fazer parte do endereço do app.
> Ex: `leandrosouza.github.io/biblia`

---

## PASSO 2 — Criar o repositório

1. Após entrar no GitHub, clique no botão **"New"** (ou **"+"** no canto superior direito → "New repository")
2. Em **"Repository name"**, digite: `biblia`
3. Deixe marcado **"Public"** (obrigatório para o GitHub Pages gratuito)
4. Marque a opção **"Add a README file"**
5. Clique em **"Create repository"**

---

## PASSO 3 — Enviar os arquivos

Agora você vai enviar os arquivos da Bíblia para o GitHub.

### 3.1 — Abrir a página de upload

1. Na página do seu repositório, clique em **"Add file"**
2. Clique em **"Upload files"**

### 3.2 — Selecionar os arquivos

Arraste ou selecione **todos** os arquivos e pastas que você recebeu:

```
index.html
style.css
script.js
manifest.json
sw.js
logo_biblia.png
icons/          ← pasta inteira
Biblias/        ← pasta inteira com todos os .json
```

> ⚠️ **Atenção com as pastas:** O GitHub não aceita pastas arrastadas em alguns navegadores.
> Se não conseguir enviar as pastas, use o método alternativo abaixo.

### 3.3 — Método alternativo para pastas (se necessário)

Se o navegador não aceitar as pastas:

1. Selecione e suba primeiro os arquivos soltos (`index.html`, `style.css`, etc.)
2. Depois clique em **"Add file" → "Upload files"** novamente
3. Entre na pasta `Biblias/` do seu computador e selecione todos os `.json`
4. Antes de confirmar, clique em **"Upload files"** e no campo de destino coloque `Biblias/`

Ou, mais simples: **compacte em um ZIP e use o método GitHub.dev** (veja abaixo).

### 3.4 — Confirmar o envio

Depois de selecionar os arquivos:
1. Role a página para baixo
2. Clique em **"Commit changes"**
3. Aguarde o upload (pode demorar alguns minutos por causa dos arquivos JSON grandes)

---

## PASSO 4 — Ativar o GitHub Pages

1. Na página do repositório, clique em **"Settings"** (Configurações)
2. No menu lateral esquerdo, clique em **"Pages"**
3. Em **"Branch"**, selecione **"main"**
4. Clique em **"Save"**

Aguarde cerca de 1-2 minutos. O GitHub mostrará:

> ✅ **"Your site is published at https://SEU_USUARIO.github.io/biblia/"**

Esse é o endereço do seu app! Compartilhe com seus amigos.

---

## PASSO 5 — Instalar no celular (você e seus amigos)

1. Abra o **Chrome** no celular Android
2. Acesse o endereço: `https://SEU_USUARIO.github.io/biblia/`
3. Aguarde carregar (na primeira vez baixa tudo — pode demorar pelo tamanho dos JSONs)
4. Aparecerá um aviso na tela: **"Instalar Bíblia Sagrada como aplicativo"**
5. Toque em **"Instalar agora"**
6. Pronto! O ícone aparece na tela inicial do celular

**Da próxima vez, abre sem internet, como um app normal.**

---

## ❓ Dúvidas Frequentes

**Quanto tempo o site fica no ar?**
Para sempre, enquanto você mantiver a conta no GitHub (é gratuito).

**E no iPhone funciona?**
Sim! No Safari, toque no ícone de compartilhar → "Adicionar à Tela de Início".

**Posso mandar o link pelo WhatsApp?**
Sim! É só mandar o link `https://SEU_USUARIO.github.io/biblia/` e qualquer pessoa pode instalar.

**E se eu quiser atualizar depois?**
Basta voltar ao GitHub, clicar no arquivo que quer trocar e editar ou fazer novo upload.
